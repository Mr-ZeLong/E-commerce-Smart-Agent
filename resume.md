# 电商智能客服 Agent 系统

## 项目描述
主导设计基于 **LangGraph + 大语言模型** 的电商智能客服 Agent 系统，解决传统客服人工成本高、响应慢、风险管控难的痛点。采用 **Multi-Agent 协作架构**，整合 RAG 检索增强、多维度置信度评估、智能风控分级审核与人工接管机制，实现订单查询、政策咨询、退货申请等核心场景的端到端自动化处理，日均处理数千会话，显著降低人工介入率。

## 项目技术
**LangChain** | **LangGraph** | **FastAPI** | **SQLModel** | **PostgreSQL + pgvector** | **Redis** | **Celery** | **LLM API** | **Pydantic** | **JWT**

## 职责描述

### Agent 编排与多 Agent 协作架构
- 设计 **三层 Agent 架构**：RouterAgent（意图路由层）→ SupervisorAgent（协调监督层）→ Specialist Agents（领域专家层：订单/政策 Agent）
- 基于 **LangGraph 状态机** 实现工作流编排：定义意图识别、知识检索、订单查询、退货处理、资格审核、回复生成六大核心节点，支持状态持久化与断点恢复
- 实现动态路由决策：规则前置过滤 + LLM 意图识别双层策略，支持订单查询/政策咨询/退货申请/闲聊等多类意图精准分发，有效减少 LLM 调用开销
- 封装 **BaseAgent 抽象基类**，定义标准化 Agent 接口与生命周期管理，支持依赖注入与 Mock 测试

### RAG 检索增强与知识库系统
- 实现基于 **pgvector** 的语义检索：自定义 Embedding 适配器兼容主流向量化 API，支持高维向量存储与相似度计算
- 设计 **动态阈值过滤机制**，基于检索质量自适应调整过滤策略，有效抑制低质量检索导致的模型幻觉
- 构建电商政策知识库 ETL 流程：支持多格式文档（Markdown/PDF）自动分块、向量化、索引构建与增量更新
- 多源上下文融合：Policy 知识检索与 Order 业务数据动态组装，构建完整上下文供生成节点使用

### 置信度评估与质量保障
- 设计 **多维度置信度信号系统**：
  - **RAG Signal**：基于检索相似度分布、查询-内容覆盖率等指标评估检索质量
  - **LLM Signal**：模型自评估回答置信度，带重试机制与健壮性解析适配多种输出格式
  - **Emotion Signal**：用户情感状态实时检测（挫败感/紧急度/满意度），识别高风险交互场景
- 实现 **智能成本优化策略**：信号明确场景下动态跳过冗余评估节点，在保证质量的同时降低 API 调用成本
- 配置化阈值管理：支持多级别审核阈值与权重配置，可根据业务场景灵活调整策略

### 智能风控与人工审核
- 设计 **分级风控规则引擎**：基于退款金额、用户等级、历史行为等多因子划分风险等级，实现低风险自动处理、中高风险人工审核的分级策略
- 实现 **人机协作闭环**：高风险订单自动创建审计日志，触发异步任务通知运营团队，WebSocket 实时推送用户审核状态，支持一键决策与会话上下文回放
- 完整的审计追踪体系：Context Snapshot 记录完整决策上下文与元数据，支持事后分析、合规审计与模型效果评估

### 异步任务与实时通信
- 基于 **Celery + Redis** 构建高可靠异步任务队列：处理退款支付、短信通知、运营告警等耗时操作，支持任务重试与死信队列
- 实现 **WebSocket 连接管理器**：支持多终端实时状态同步、心跳保活、断线重连与连接池管理
- FastAPI SSE 流式响应：大模型生成内容逐字流式推送，降低首字时延，提升交互体验
